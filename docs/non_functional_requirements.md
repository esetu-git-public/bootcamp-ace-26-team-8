# Non-Functional Requirements

## Accuracy

The system should provide reliable loan default predictions with acceptable performance based on evaluation metrics such as Accuracy, Precision, Recall, and F1-Score.

## Performance

The system should generate prediction results within a few seconds after receiving valid borrower information.

## Reliability

The system should operate consistently with minimal failures and produce dependable prediction results.

## Scalability

The system should support larger datasets and an increasing number of prediction requests without significant performance degradation.

## Maintainability

The application should be modular and easy to update, allowing improvements to the machine learning model and system components with minimal effort.

## Usability

The system should provide a simple and intuitive interface that enables users to enter borrower details and view prediction results easily.

## Security

Borrower information should be protected from unauthorized access, and sensitive data should be handled securely.

## Availability

The system should be available whenever required by authorized users and maintain stable operation during normal usage.

## Portability

The application should be deployable on different operating systems such as Windows and Linux with minimal configuration changes.
